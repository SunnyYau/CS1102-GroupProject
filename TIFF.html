TIFF

What is TIFF file format and What compression algorithms used:

Tiff is a kind of lossless image format. It is primarily used to transfer images between two devices and high-quality printing since it could remain a high resolution and quality.

Lossless compression algorithm makes files smaller and faster to transfer. Also, it allows the original file to be intact if the file is compressed and decomposed back to the original version. Unlike JPGs and PNGs, TIFF is uncompressed.

But when the TIFF file is huge, many compression algorithms can be utilized to reduce storage cost such as LZW and ZIP algorithms. In application, both LZW and ZIP offer similar compression.

How does LZW compression algorithm works:

LZW compression algorithm works by reading a sequence of symbols and constructing a table called codebook with a particular bit pattern set.The number of table entries usually is 4096, the first 256 (0-255) codes are allocated to standard character sets and the rest of the table is blank.

When process it, the table keeps expanding with new characters. If the same pattern on the table appears again in the input, a shorter code replaces that pattern. Shorter codes occupy less space than the original pattern. With the increase of shorter codes, the algorithm is also efficient since the simpler file.

Pros of LZW compression

  • Efficiency:
    It is useful for files containing plentiful repetitive data and is commonly useful with monochrome images.
  • High compression ratio:
    Since this algorithm applies dynamic compression of the data being compression, so the compression ratio can be high enough despite the repetitive data
  • Simple to operate:
    It is relatively simple to use as prior input about the data flow is not required.
  • Reliable algorithm:
    LZW supports most operating systems and software applications. So it is widely used as a ideal choice for compression and decompression

Cons of LZW compression

  • Cost:
    It was patented in the 80s, so licensing fees are required for many years. It may increase the extra cost to the application of LZW.
  • Speed:
    it is slower than other compression algorithms since it has to keep updating its codebook. Hence, LZW is not the most efficient compression algorithm unless the file contains a lot of repeated data.
  • LZW vs ZIP:
    It is available to both LZW and ZIP algorithms for 8-bit TIFF files. However, the ZIP algorithm is a better option than LZW for 16-bit TIFF files because LZW may make 16-bit files larger
